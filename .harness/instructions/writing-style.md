# Writing style

This applies to every document, plan, and commit message the harness produces.

## Plain English

- Write the way you would explain something to a colleague at their desk.
- Short sentences. Common words. Say the thing directly.
- If a sentence does not add information, delete it.

## No AI fluff

Cut all of this:

- "In today's fast-paced world..."
- "It's important to note that..."
- "Let's dive in!"
- "This robust, comprehensive, powerful solution..."
- Emoji used as decoration.
- Restating the heading in the first sentence.

## Be current, not historical

- Documents describe how things are now. Not how they used to be.
- Do not write "we changed X to Y" or "originally this did Z". The page reflects the present only.
- The exception is plans, which capture intent at a point in time and then lock. Plans are allowed to say "we will do X". Concepts, decisions, and specs are not.

## Structure

- Lead with the point. Details after.
- Use headings and short lists so a human can scan in seconds.
- One idea per paragraph.

## Test for a good document

Read it out loud. If it sounds like a person explaining something useful, it is good. If it sounds like a brochure or a tutorial intro, rewrite it.
