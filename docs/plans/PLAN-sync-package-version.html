Sync package version

Bump package.json to 1.1.0 so the published package matches the v1.1.0 git tag.

implemented version release npm
Created 2026-07-03 · Implemented 2026-07-03

Context

package.json reads 0.1.0 while the release tags are at v1.1.0. A publish would ship the wrong version and npx continuity-harness would not match the tagged release.

Approach

Set version in package.json to 1.1.0.

Files

  • package.json — version field.

Verification

node -p "require('./package.json').version" prints 1.1.0, matching git tag.
