Bootstrap the Engineering Continuity Harness

Stand up the full harness: engine, hooks, knowledge docs, and the npx installer.

implemented harness bootstrap hooks installer
Created 2026-06-28 · Implemented 2026-06-28

Context

This project is a coding-agent-agnostic harness. It keeps agents like Claude disciplined and keeps a project's history and knowledge readable by humans over time. This first plan builds the whole system, and the harness records its own creation as its first plan.

Approach

Two halves: .harness/ is the engine and ships with the installer; docs/ is the project's own output and is never shipped. The .claude/ folder is a thin pointer plus hook wiring.

  • Rules and instructions in .harness/rules and .harness/instructions.
  • HTML templates for plans, concepts, decisions, specs.
  • A Node index generator that builds the master hub and the plan navigator.
  • Four hooks: inject rules at session start, lock resolved plans and require a plan before code, keep work off main, and rebuild the index when docs change.
  • An npx installer with init and update.

Files

  • .harness/ — README, rules, instructions, templates, bin (build-index, hooks, libs).
  • .claude/CLAUDE.md, .claude/settings.json.
  • package.json, bin/cli.mjs.
  • docs/ — generated indexes and this plan.

Verification

  • Run the index generator and open docs/index.html and docs/plans/index.html.
  • Pipe sample hook payloads into each hook and confirm allow/deny behavior.
  • Run the installer into a scratch directory and confirm the engine lands and docs are created.
