Make the harness a well-behaved, self-integrating citizen

Rebuild the installer so the harness merges into each agent's native config instead of parking sidecars, and heals itself when another tool overwrites it.

implemented installer merge integration self-heal
Created 2026-07-04 · Implemented 2026-07-04

Context

The harness drops into a repo and wires itself into whatever agent tooling is present (Claude, Cursor, Codex, Antigravity) plus git. Today it is not a good citizen: when a config file already exists, the installer parks a sidecar (settings.harness.json) and asks the developer to merge by hand, and for git it gives up if core.hooksPath is already taken and otherwise hijacks the whole hooks path.

Graphify — the tool being evaluated alongside — is the model to follow. It merges into the native files: it preserves every existing key, strips only its own prior entries, appends its own, and writes back. Its git installer respects an existing core.hooksPath, validates the path stays in the repo, and marker-appends to an existing hook rather than clobbering it. It also ships a clean uninstall.

The outcome: the harness blends into each agent's native config as a first-class citizen, is idempotent and cleanly removable, and is resilient — a third-party tool touching the same files cannot silently disable it. Where graphify silently resets settings to an empty object on a JSON parse error, the harness is safer: it backs up and warns, never wipes.

Approach

  • New shared, fail-safe merge library .harness/bin/lib/merge.mjs: safe JSON read (never returns empty on parse failure), hook-array merge that keeps user and foreign entries and replaces only harness entries, permission-array union, marker-section replace, hooks-dir resolution honoring an existing core.hooksPath, and marker-based hook install/removal.
  • Rewrite bin/cli.mjs integrators to merge into .claude/settings.json, .cursor/hooks.json, .codex/hooks.json, .agents/hooks.json instead of parking sidecars. Route CLAUDE.md / AGENTS.md through the marker-replace helper so instruction edits propagate on update. Keep overwrite only for the uniquely-named harness rule files.
  • Stop hijacking git. Resolve the active hooks dir and marker-append the harness commit-msg check into it, so it coexists with Husky, graphify, and user hooks and no longer shadows .git/hooks.
  • Add an uninstall command that reverses every merge via markers and identity, leaving all user and foreign content intact.
  • Extend the SessionStart hook to verify and repair the harness wiring, so another installer stripping it is restored on the next session.
  • Update .harness/README.md to describe merge-not-sidecar, coexistence, uninstall, and the per-clone update note.

Files

  • New: .harness/bin/lib/merge.mjs.
  • Rewrite: bin/cli.mjs (merge integrators, guest git install, uninstall command).
  • Edit: .harness/bin/hooks/claude/session-context.mjs (verify and self-heal).
  • Edit: .harness/README.md and root README.md.
  • Reused as-is: .harness/integrations/** payloads and .harness/githooks/commit-msg.mjs.

Verification

  • Seed .claude/settings.json with a user hook and graphify's hooks, run the installer, and confirm all entries survive, the harness entries are added, the file is valid JSON, and a second run is byte-identical.
  • Pre-seed a Husky-style commit-msg, install, and confirm the harness block is appended not clobbered; a bad commit message is rejected and a good one passes.
  • Install the harness and graphify into the same repo in both orders and confirm both survive.
  • Feed invalid JSON to the installer and confirm it backs up and warns rather than wiping.
  • Run uninstall and confirm harness blocks are removed while user and foreign content stay.
  • Delete the harness hook entry, start a session, and confirm SessionStart restores it.
